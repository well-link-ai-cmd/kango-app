# 訪問看護3様式 AI自動生成ツール 手順書

> 責任者レビュー用ドキュメント。厚労省通知・2024年度改定・日本訪問看護財団の記載要領に準拠した作成手順。この手順をもとにAIプロンプトを設計する。

---

## 0. 全体の法的根拠

- 厚労省通知「**保医発0327第2号（令和2年3月27日）**」で標準様式が定められている
- **別紙様式1/2** = 通常の訪問看護計画書/報告書
- **別紙様式3/4** = 精神科訪問看護計画書/報告書
- 報告書は**主治医へ毎月提出義務**（訪問看護管理療養費の算定要件と連動）
- **2024年度改定の要点**：
  - 訪問看護管理療養費が「1」「2」に区分
  - 指示書に「傷病名コード」欄追加
  - **精神科訪問看護はGAF尺度の記載が必須化**（記載漏れは返戻リスク）

---

## 1. 訪問看護報告書（通常・別紙様式2）

### 1-1. 法的要件
- 根拠：保医発0327第2号、令和6年度改定
- 提出先：**主治医（月1回）**
- 算定連動：訪問看護管理療養費（1または2）

### 1-2. 各欄の書き方ルール

| 欄 | 書くべき内容 | ポイント |
|---|---|---|
| 病状の経過（3000字） | バイタル変化・症状変動・治療反応・ADL変化・計画目標に対する評価 | 報告書で最も文章量が多い。冗長化禁止、問題点と解決策を具体的に |
| 看護・リハの内容（3000字） | 実施ケアを**箇条書き推奨**（清潔ケア・服薬管理・創傷処置等） | 簡潔に |
| 家庭での介護の状況（3000字） | **誰が・どのように**介護しているか、介護者の心身状態・介護負担の安定性 | キーパーソン明記 |
| 衛生材料等の種類・量 | 使用量・過不足・変更必要性（主治医への依頼根拠） | 指示書への反映を意識 |
| 特記すべき事項（3000字） | 他欄に収まらない重要事項、頻回訪問理由 | |
| 訪問日暦 | **看護師=○、PT/OT/ST=◇、特別指示書=△** | 記号ルール厳守 |

### 1-3. リハ別添（PT/OT/STが訪問した場合のみ）
- **障害高齢者の日常生活自立度**：J1/J2/A1/A2/B1/B2/C1/C2
- **認知症高齢者の日常生活自立度**：自立/Ⅰ/Ⅱa/Ⅱb/Ⅲa/Ⅲb/Ⅳ/M
- **Barthel Index（10項目・100点満点）**
- コミュニケーション能力

### 1-4. AI責任分界
- ✅ **AI生成OK**：病状経過の下書き（SOAP集約→時系列要約）、看護内容の箇条書き化、衛生材料使用量集計
- ❌ **看護師判断必須**：Barthel Index点数・自立度ランク判定、計画目標評価、主治医への依頼事項、特記事項の優先順位

## 2. 訪問看護報告書（精神科・別紙様式4）

### 2-1. 法的要件
- 根拠：保医発0327第2号、2024年度改定
- 算定連動：**精神科訪問看護基本療養費（Ⅰ）または（Ⅲ）**
- 対象：精神疾患を有する利用者
- **2024改定でGAF記載が必須化**（記載漏れは返戻リスク）

### 2-2. 各欄の書き方ルール

| 欄 | 書き方 |
|---|---|
| 病状の経過 | 精神症状の推移、服薬状況、睡眠/食欲/対人交流、**希死念慮等のリスク**を明示 |
| 看護内容 | 服薬支援、症状モニタリング、生活リズム支援、SST等 |
| **家族等との関係** | 家族構成、キーパーソン、関係性の良好/緊張、家族の疾病理解、家族支援の要否 ※通常様式にはない精神科特有項目 |
| 特記事項・GAF尺度 | 「10 GAF」を○で囲み、**当該月の初日訪問時**の判定値と判定年月日を記載 |
| 頻回訪問理由 | 症状悪化/服薬中断リスク/自傷他害リスク等を**具体的に** |
| 情報提供先 | 保健所・市町村・医療機関等との連携状況 |

### 2-3. GAF尺度の運用ルール（最重要）
- 0〜100点、10段階区分
- **月の初日の訪問時に判定**（以降の訪問で変動しても更新しない）
- **直近1週間で最も症状が悪かったエピソード**で評価
- 判定者：精神科訪問看護の研修を修了した看護師等
- 家族のみ訪問の月：「家族への訪問看護でありGAF判定不能」と明記

### 2-4. AI責任分界
- ✅ **AI生成OK**：病状経過・看護内容の下書き、家族関係の事実記述
- ❌ **看護師判断必須**：**GAF点数判定（AI厳禁）**、頻回訪問の必要性判断、リスクアセスメント、情報提供先の選定

## 3. 訪問看護情報提供書

### 3-1. 3つの算定区分

| 区分 | 提供先 | 目的 | 対象 |
|---|---|---|---|
| **情報提供療養費1** | **市町村・都道府県**（保健所長含む） | 在宅療養の総合的促進、保健福祉サービス連携 | 別表7・8該当者、精神障害者、特掲診療料別表7の疾病等 |
| **情報提供療養費2** | **学校** | 医療的ケア児の学校生活支援 | 18歳未満の医療的ケア児 |
| **情報提供療養費3** | **医療機関・介護老人保健施設・介護医療院** | 療養場所変更時の切れ目ない看護継続 | 入院・入所する利用者 |

- 1利用者につき月1回算定

### 3-2. 宛先4種の書き分け（カイポケUIベース）

| 宛先 | トーン・重点 | 特記事項 |
|---|---|---|
| **市町村** | 生活支援・福祉サービス調整寄り。介護保険外のインフォーマル支援、経済的困窮、独居リスク | 地域包括支援センター連動 |
| **保健所長** | **公衆衛生寄り**。難病医療費、結核、感染症、精神保健、医療的ケア児支援 | 法令（難病法/精神保健福祉法）根拠を意識 |
| **学校** | **小児・教育支援寄り**。医療的ケア内容（吸引/経管栄養等）、緊急時対応、体育/行事参加の可否、投薬スケジュール | 保護者同意、養護教諭/学校看護師との連携 |
| **医療機関** | **医療連携寄り**。在宅での経過、投薬、処置、看護上の問題点、継続必要ケア | 看護師視点の記述（医師→医師の診療情報提供書とは別物） |

### 3-3. 医療機関宛 vs 診療情報提供書の違い
- **診療情報提供書**：医師→医師、診断・治療方針が中心
- **訪問看護情報提供書3**：看護師→医療機関看護部門、**在宅での生活情報・看護ケア継続性**が中心
- → 両者は補完関係。看護師は看護師視点で書く

### 3-4. AI責任分界
- ✅ **AI生成OK**：主傷病・病状の下書き、ADL項目の選択肢提示、宛先別テンプレート出力
- ❌ **看護師判断必須**：宛先選定、ADL 3段階判定、算定区分（1/2/3）の選択、個人情報開示範囲

## 4. 評価スケール詳細

### 4-1. Barthel Index（10項目・満点100）
| 項目 | 配点 |
|---|---|
| 食事 | 10 / 5 / 0 |
| 車椅子⇔ベッド移乗 | 15 / 10 / 5 / 0 |
| 整容 | 5 / 0 |
| トイレ動作 | 10 / 5 / 0 |
| 入浴 | 5 / 0 |
| 平地歩行 | 15 / 10 / 5 / 0 |
| 階段昇降 | 10 / 5 / 0 |
| 更衣 | 10 / 5 / 0 |
| 排便コントロール | 10 / 5 / 0 |
| 排尿コントロール | 10 / 5 / 0 |

- 判定目安：85点以上=自立 / 60点=部分自立 / 40点=大部分介助 / 0点=全介助
- 評価者：看護師・リハ職等、研修受講推奨

### 4-2. GAF尺度（0〜100）
| 範囲 | 状態 |
|---|---|
| 91-100 | 症状なし、広範囲活動で優れた機能 |
| 81-90 | 症状なしかごく軽度 |
| 71-80 | 心理社会的ストレス要因に対する一過性反応 |
| 61-70 | 軽度症状 |
| 51-60 | 中等度症状 |
| 41-50 | 重篤な症状 |
| 31-40 | 現実検討・コミュニケーションに欠陥 |
| 21-30 | 妄想・幻覚に影響、ほぼ全機能障害 |
| 11-20 | 自傷他害の危険 |
| 1-10 | 重度の自傷他害の危険 |
| 0 | 情報不十分 |

### 4-3. 日常生活自立度
- 寝たきり度：J1/J2（生活自立）→ A1/A2（準寝たきり）→ B1/B2（寝たきり）→ C1/C2（重度寝たきり）
- 認知症自立度：自立 → Ⅰ → Ⅱa/Ⅱb → Ⅲa/Ⅲb → Ⅳ → M（専門医療対応）

## 5. AI自動化の注意点

### 禁忌事項
- **事実創作の絶対禁止**：訪問していない日の記録、測定していないバイタル、行っていないケア
- **個人情報取扱**：主治医以外（特に医療機関外）への情報提供時は宛先と同意範囲を厳格管理
- **責任帰属**：AIは下書きのみ、**最終確認・押印は管理者/担当看護師**
- **点数判定のAI禁止**：GAF・Barthel・自立度は看護師判断必須

### AI生成OK領域
- 月次SOAPの時系列集約・要約
- 箇条書きフォーマット変換
- 文字数調整（3000字/1000字制限への圧縮）
- 宛先別トーン変換
- 記号/自立度コードの記載フォーマット整形

### 看護師判断必須領域
- 評価スケール点数判定（Barthel・GAF・自立度）
- 頻回訪問の医療必要性根拠
- 主治医への依頼事項
- 情報提供先の選定
- リスクアセスメント

## 6. AIプロンプト設計への示唆（7点）

1. **分岐軸を最初に確定**：①様式種別（通常/精神科）②リハ別添の要否 ③情報提供書宛先（市町村/保健所長/学校/医療機関）の3分岐を最初に問い、以降のプロンプトを切替える

2. **スケール値は"入力必須"として看護師に要求**：Barthel Index 10項目・GAF点数・自立度ランクはAI推定禁止。UIで看護師から入力を受けた上でプロンプトに挿入する

3. **文字数制御を二段構え**：初回生成時に上限（通常3000字/情報提供書1000字）を指示、生成後に文字数カウントして超過時は自動圧縮プロンプトを再実行

4. **事実範囲を明示的に制約**：プロンプトに「入力されたSOAPに記載のない事実は一切推論・創作しない」「不明項目は『記載なし』と明記」を必ず含める

5. **宛先別トーンを分離プロンプト化**：
   - 学校宛＝「医療的ケアと学校生活配慮を小児支援の視点で」
   - 保健所長宛＝「公衆衛生・難病医療費公費負担の視点で」
   - 市町村宛＝「地域包括支援・福祉サービス連携の視点で」
   - 医療機関宛＝「在宅継続ケアと看護上の問題点を看護申し送りの視点で」

6. **記号/コードの機械処理**：訪問日暦（○◇△）、GAF「10」を○で囲む表記、傷病名コード等はテンプレート挿入で自動処理し、自由文生成と分離

7. **"責任者レビュー欄"を必ず出力に含める**：AI生成箇所には「※AI下書き。看護師確認必須」マーカーを付し、GAF/Barthel/自立度欄は空欄にして看護師記入を促す構造にする

## 7. 実装時のDB追加項目（案）

```
visit_reports テーブル
- id
- patient_id
- report_type         -- 'normal' | 'psychiatric'
- target_month        -- YYYY-MM
- created_at, updated_at
- disease_progress    -- 病状の経過（3000字）
- nursing_content     -- 看護の内容（3000字）
- family_care         -- 家庭での介護の状態 or 家族等との関係（3000字）
- hygiene_material    -- 衛生材料（JSON）
- special_notes       -- 特記すべき事項（3000字）
- visit_calendar      -- 訪問日暦（JSON: {date, symbol}[]）
- rehab_attachment    -- リハ別添（JSON、通常のみ）
  - daily_life_level  -- J1〜C2
  - dementia_level    -- 自立〜M
  - barthel_index     -- JSON（10項目、看護師入力）
  - barthel_total     -- 合計点
  - communication     -- 文字列（3000字）
- gaf_score           -- GAF点数（精神科のみ、看護師入力）
- gaf_judge_date      -- GAF判定日
- gaf_unavailable     -- 家族訪問でGAF判定不可フラグ

info_provisions テーブル
- id
- patient_id
- addressee_type      -- 'municipality' | 'health_center' | 'school' | 'medical_institution'
- fee_category        -- 1 | 2 | 3（情報提供療養費区分）
- target_month
- adl_6items          -- JSON（6項目、看護師入力）
- main_disease        -- 主傷病名（1000字）
- disease_state       -- 病状・障害等の状態（1000字）
- monthly_visits      -- {days, count}
- family_info         -- 家族等情報（1000字）
- nursing_content     -- 看護の内容（1000字）
- welfare_services    -- 必要な保健福祉サービス（1000字）
- other_notes         -- その他特筆事項（1000字）
```
