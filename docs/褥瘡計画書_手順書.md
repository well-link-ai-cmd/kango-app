# 褥瘡計画書 AI自動生成ツール 手順書

> 責任者レビュー用ドキュメント。厚労省様式・日本褥瘡学会ガイドラインに準拠した褥瘡計画書作成手順をまとめたもの。この手順をもとにAIプロンプトを設計する。

---

## 0. 前提：使用する様式

- **訪問看護では「褥瘡対策に関する看護計画書」を使う**（病院の「診療計画書」とは別物）
- 厚労省通知「保発0305第12号（令和6年3月5日）」の別紙様式
- 訪問看護管理療養費・専門管理加算（月250単位、2024年新設）の算定要件に関わる
- **本ツールはカイポケ（介護ソフト）転記前提**。カイポケの画面構成に完全準拠して出力する

## 0-1. 作成対象ルール（カイポケ画面注記に準拠）

> **日常生活自立度が正常〜A2の利用者については当該評価票の作成を要しない**

- **B1以上（B1/B2/C1/C2）で作成必須**
- A2以下は作成不要（AIはその旨を通知して処理を止める）
- B1以上で、**危険因子が1つでも「あり／できない」**なら看護計画立案が必須

---

## 1. 作成フロー（全体像）

```
患者情報 + SOAP記録
  ↓
Step 1: 日常生活自立度の判定（J1〜C2）
  ↓ A2以下 → 作成不要（終了）
  ↓ B1以上
Step 2: 危険因子の評価（7項目）
  ↓ 1つでも該当あり → 計画立案必須
Step 3: リスクアセスメント（OHスケールで点数化）
  ↓
Step 4: 褥瘡の有無・状態評価
  ↓ ありの場合 DESIGN-R®2020 は看護師が手入力
Step 5: 看護計画立案（4カテゴリ：AI生成ドラフト）
  ↓
Step 6: 看護師レビュー・修正 → カイポケに項目別コピペ
  ↓
Step 7: 評価サイクル設定（2週間〜1ヶ月後）
```

## 2. 厚労省様式の全項目

### 2-1. 基本情報
- 氏名／性別／年齢／計画作成日／記入看護師名

### 2-2. 褥瘡の有無
- 現在：なし / あり（部位：仙骨部・坐骨部・尾骨部・腸骨部・大転子部・踵部・その他）
- 過去：なし / あり

### 2-3. 日常生活自立度
`J1 / J2 / A1 / A2 / B1 / B2 / C1 / C2` の10区分

### 2-4. 危険因子評価（二値）
| 項目 | 評価 |
|---|---|
| 基本的動作能力（ベッド上の自力体位変換） | できる / できない |
| 基本的動作能力（イス上の座位保持・除圧） | できる / できない |
| 病的骨突出 | なし / あり |
| 関節拘縮 | なし / あり |
| 栄養状態低下 | なし / あり |
| 皮膚湿潤（多汗・尿失禁・便失禁） | なし / あり |
| 皮膚の脆弱性（浮腫・スキン-テア保有/既往） | なし / あり |

**→ 1項目でも「あり／できない」なら看護計画立案が必須**

### 2-5. 褥瘡の状態評価
DESIGN-R®2020 の7項目（下記）

### 2-6. 看護計画（カイポケ実装は4カテゴリ）
カイポケは以下の4カテゴリ構成（各1000字以内）。**「患者・家族教育」は独立カテゴリではなく、各項目の文末に織り込む**。

| カテゴリ | サブ区分 | 文字数 |
|---|---|---|
| 圧迫・ズレ力の排除 | ベッド上 | 1000字 |
| 圧迫・ズレ力の排除 | イス上 | 1000字 |
| スキンケア | - | 1000字 |
| 栄養状態改善 | - | 1000字 |
| リハビリテーション | - | 1000字 |

### 2-7. カイポケ画面タブ構成（実物）
`利用者情報 / 指示書と主治医 / 看護療養の情報 / 褥瘡の有無 / 危険因子の評価 / DESIGN-R2020 / 看護計画`

### 2-8. カイポケの「雛形機能」を活用する
カイポケ各記入欄に「**雛形引用 / 雛形登録**」ボタンがある。
→ **AI出力をコピペして雛形登録すれば、次回以降の使い回しが可能**。これが本ツールの資産化ルート。

## 3. リスクアセスメントスケール

### 3-1. OHスケール（訪問看護の第一選択）
- 4項目：①自力体位変換能力 ②病的骨突出 ③浮腫 ④関節拘縮
- 合計 0〜10点
- **分類**：0点=なし / 1〜3点=軽度 / 4〜6点=中等度 / 7〜10点=高度
- 短時間で評価可能

### 3-2. ブレーデンスケール（病院標準）
- 6項目：知覚の認知・湿潤・活動性・可動性・栄養状態・摩擦とずれ
- 合計 6〜23点
- **カットオフ：在宅17点以下 / 病院14点以下でリスクあり**

### 3-3. K式スケール在宅版（独居・老々介護向け）
- 前段階要因＋引き金要因
- **介護力評価が組み込まれている**のが特徴

### 推奨運用
- メイン：**OHスケール**
- サブ：**K式在宅版**（独居・老々介護ケース）
- 参考：ブレーデン（病院出身スタッフとの共通言語）

## 4. DESIGN-R®2020 採点表

| 項目 | 軽症（小文字） | 重症（大文字） | 合計算入 |
|---|---|---|---|
| **D** 深さ | d0/d1/d2 | D3/D4/D5/**DDTI**/DU | × 別記 |
| **E** 滲出液 | e0 / e1 / e3 | E6（1日2回以上交換） | 〇 |
| **S** 大きさ（cm²） | s0 / s3(<4) / s6(<16) / s8(<36) / s9(<64) / s12(<100) | S15(≥100) | 〇 |
| **I** 炎症・感染 | i0 / i1 | **I3C（臨界的定着疑い）** / I3 / I9 | 〇 |
| **G** 肉芽 | g0 / g1(≥90%) / g3(50-90%) | G4(10-50%) / G5(<10%) / G6(なし) | 〇 |
| **N** 壊死組織 | n0 | N3（柔らかい） / N6（硬い） | 〇 |
| **P** ポケット（cm²） | p0 | P6(<4) / P9(<16) / P12(<36) / P24(≥36) | 〇 |

- **合計点：0〜66点**（Dを除く）
- D は別記（重症度指標）

### 2020年改訂の重要ポイント
- **DDTI（深部損傷褥瘡疑い）** 追加 → 表面は紫/栗色でも深部進行中
- **I3C（臨界的定着疑い）** 追加 → バイオフィルム示唆（ぬめり・脆弱肉芽・滲出液多）

## 5. 看護計画（カイポケ4カテゴリに沿った標準介入内容）

### ① 圧迫・ズレ力の排除：ベッド上
- 体位変換：従来の2時間毎にエビデンスなし。粘弾性フォーム/エアマットレス使用時は**4時間を超えない範囲**で個別化
- 30度側臥位（殿筋で荷重）
- スモールチェンジ（微小位置変更）の活用
- 体圧分散寝具の選定（OHスケール点数に応じて）
- **家族指導**：体位変換手技・除圧タイミングの指導をこの文中に含める

### ② 圧迫・ズレ力の排除：イス上
- 車椅子・座位時の15分ごとプッシュアップ（家族代行含む）
- 座圧分散クッション選定
- 座位姿勢の保持方法・ずれ防止
- **家族指導**：座位時の観察・声かけをこの文中に含める

### ③ スキンケア
- 弱酸性洗浄剤で愛護的に洗浄（擦らない）
- 保湿剤（ヘパリン類似物質等）
- IAD予防：撥水性クリーム
- スキン-テア予防：長袖・アームカバー
- **家族指導**：清拭・入浴時の観察ポイント（発赤持続・水疱）をこの文中に含める

### ④ 栄養状態改善
- **MNA-SF（6項目/14点）**: 0〜7=低栄養 / 8〜11=低栄養のおそれ / 12〜14=良好
- **Alb 3.5 g/dL以下でリスク上昇**
- 治癒期目標：**30 kcal/kg/日以上、たんぱく質1.0 g/kg/日以上**（重症時1.2〜1.5）
- 亜鉛・アルギニン・ビタミンC補給
- **家族指導**：食事量・水分量の記録方法をこの文中に含める

### ⑤ リハビリテーション
- 関節可動域訓練（拘縮予防）
- 座位耐性訓練・離床プログラム
- PT/OT/STとの連携
- **家族指導**：日常での可動域維持の工夫をこの文中に含める

### ドレッシング材選択フロー
| 状態 | 選択 |
|---|---|
| 滲出液少 (e1) + 感染なし | ハイドロコロイド |
| 滲出液中〜多 (e3/E6) | ポリウレタンフォーム / アルギン酸塩 / ハイドロファイバー |
| 感染あり (I3/I3C) | **閉鎖性ドレッシング禁忌**。外用薬（カデキソマー・ヨウ素等）で感染制御優先 |
| 壊死組織 (N3/N6) | デブリードマン |

## 6. 評価サイクル

- **推奨見直し頻度：2週間〜1ヶ月ごと**
- 評価指標：DESIGN-R®2020推移／危険因子変化／目標達成度／生活環境変化
- 悪化時は医師へ即報告
- **毎年8月に褥瘡保有者数を地方厚生局へ報告義務**

---

## 7. AI自動化の責任分界点（最重要）

### ✅ AIにやらせていい（記録要約系）
- SOAP記録からの危険因子該当抽出（例：「自力体位変換困難」→「基本的動作能力: できない」）
- 看護計画5軸の標準介入文ドラフト生成
- 過去記録からのDESIGN-R推移要約
- 目標設定（SMARTゴール）叩き台

### ❌ AIに任せてはダメ（看護師が必ず判断）
- **DESIGN-Rの採点**（画像・触診・滲出液性状の直接観察必須）
- **感染判定（I3・I3C）**（臨床所見の統合判断）
- **ドレッシング材・外用薬の最終選択**（医師指示が前提）
- **DTI疑いの判断**（触診所見必須）
- **最終計画の妥当性判断・サイン**（法的責任は看護師）

### UI設計の原則
- 画面上で「AIドラフト」と「看護師確認必須項目」を**色分け・枠分け**で明示
- DESIGN-Rは**手入力欄のみ**（AIは採点しない）
- 危険因子抽出は「記録に記載あり」「記録に記載なし（未評価）」を区別

---

## 8. AIプロンプト設計の要点（5つ）

1. **二段構え出力**：「AIドラフト」と「看護師確認必須」を明示分離
2. **厚労省様式の項目構造に完全準拠したJSONスキーマをプロンプト内に埋め込む**（Enum化、自由記述禁止）
3. **「記録にない情報は推測で埋めない」を厳命**。未記載は「未評価（要確認）」で出す
4. **5軸テンプレート＋リスク連動の条件分岐**（OHスケール点数・DESIGN-R項目別点数で介入文差し替え、エビデンス出典を併記）
5. **評価サイクル内蔵**：出力文末に「次回評価日（作成日+2週間）」と「評価チェックリスト」を自動付与

## 9. 実装時のDB追加項目（案）

```
pressure_ulcer_plans テーブル
- id
- patient_id
- created_at, updated_at
- next_review_date   -- 次回評価日
- staff_name         -- 記入看護師名
- staff_title        -- 肩書き
- daily_life_level   -- J1〜C2
- risk_factors       -- JSON（7項目の評価）
- oh_scale_score     -- OHスケール点数
- has_current_ulcer  -- 現在の褥瘡有無
- current_locations  -- 現在の部位（JSON配列）
- current_onset_date -- 発生日
- has_past_ulcer     -- 過去の褥瘡有無
- past_locations     -- 過去の部位（JSON配列）
- past_healed_date   -- 治癒日
- design_r           -- DESIGN-R採点（JSON、看護師手入力）
- plan_bed           -- 圧迫・ズレ力：ベッド上（1000字以内）
- plan_chair         -- 圧迫・ズレ力：イス上（1000字以内）
- plan_skincare      -- スキンケア（1000字以内）
- plan_nutrition     -- 栄養状態改善（1000字以内）
- plan_rehab         -- リハビリテーション（1000字以内）
- evaluation_notes   -- 評価記録
```
